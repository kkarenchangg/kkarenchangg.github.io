Houzz Pro - Website Services Pilot Program
Houzz Site Designer, Figma, Adobe XD

Overview

Over 3 months, I laid down the foundation of the Houzz Website Services Pilot Program, which included the visual design process, creation of over 30 custom websites, consultations with clients, and product design on Site Designer, a tool built by Houzz to build websites for their Pro's.
‍
Who are Houzz Pro's? Houzz Pro's are any interior designers, contractors, consultants, etc. who want to advertise their projects and services to the Houzz platform for homeowners to find them. Houzz Pro offers services created to provide one hub for all the needs that a Pro has when starting a project with a client — including invoicing, project management, and website services, which is the team I focused on. Houzz Pro also offers the service of a website specialist on the Website Services team, which was my role, to assist in creating the Pro's site. Though Houzz Pro offers this service, Pros are welcome and encouraged to use the Site Designer tool themselves.

Goals

Initially, there was a lot of back and forth between pros each time they needed an edit, and the satisfaction level of the websites created was quite low. The ultimate goal of this service was to create beautiful custom sites the first time around, and for each pro to have confidence in what they presented to their users. As a team, we wanted to reach the following goals:
     - Improve the tool to create an experience that was intuitive for a non-technical end user when creating their own site
     - Achieve a minimum Pro satisfaction rating of 9 out of 10 for at least 80% of the sites created
     - Reduce creation process of Website Services to a maxiumum of 2 hours for each account

Targeted Audiences

The Houzz Site Designer is mainly targeted toward Pro's, who generally do not have much technical experience in web design or development. The tool limits some functions to have a few options in order to maintain consistency throughout the site — for example, automatic text options for titles and paragraphs rather than the option to customize every text box. Website Specialists and internal Houzz representatives also use this tool, but mainly to create and edit sites for Pro's. They have a strong understanding of the tool, but not necessarily of web design or HTML/CSS structures.

Visual design

Using Houzz's Site Designer website builder tool, I created unique websites for each Houzz Pro. To ensure Pro satisfaction, I created an onboarding design brief form to send to each Pro in order to provide a space where they could express any preferences they had for their websites. This began a conversation to help me not only create a site that would be closest to the pro’s preferences, but also to own the relationship I had with each pro I worked with.

Building Websites

I created sites for each pro using a hybrid of templates offered on the Site Builder tool and custom aspects based on the Pro's needs. This significantly increased the pro’s satisfaction rating with their sites, and decreased the need for the initial back and forth discussions needed with the Pros before the consultations took place through this pilot program.

Over the course of three months, satisfaction levels sky rocketed and complaints diminished. The involvement with the Pro increased 153% before creating the site as a result of the onboarding form I created to collect brand elements and styles, ensuring that any site designer in the future will also now have a much clearer understanding of the Pro's preferences.

Consultations with Clients

Because each client has unique brand guidelines, I dedicated an hour to each pro to present to them their new site and explain my intentions behind each choice I made. This opened the opportunity to provide explanations for web design choices based on the Pro's needs, and also for the Pro to offer feedback along the way. By having the experience of talking to the end users directly, I reduced the need for countless hours of unnecessary email exchanges from unsatisfied Pros who wanted to make changes to their sites. Based on the pro feedback, their satisfaction levels were at a maximum with little to no post-consultation edits.

HOUZZ SITE DESIGNER - CONTEXTUAL EDITING TOOLBAR

Created with Figma
Product Design
‍
Because I worked so closely with Pro's through consultation meetings and used the Site Designer myself daily, I was able to provide insight to the engineering team on how users would approach and navigate the tool from our Pro's. Based on user feedback and engagement metrics, I found an increase in back and forth between the site preview and text editor, and decided to make prototypes for a redesign which would allow the user to make changes to the text/images while seeing those changes be made live in the context of their site. This is anticipated to be much more efficient and intuitive than the previous design of the tool, in which the modals would only allow for changes to be made for blocks on hover before seeing those changes in the site’s context (see first image).

Before - Editor blocks on hover to change text, but no way to view the changes being made live
After - Live edits can be made and seen as the user clicks on each block

Thoughts / What I learned

This update to the tool can be a huge improvement to the current user experience by eliminating the need to imagine an edit before seeing it happen live. Based on the poor user engagement from the previous design and the feedback received from our customers, we expect the usage of this feature to increase. This redesign allows users to view changes on demand, ultimately eliminating unnecessary clicks and creating a more seamless experience.

I really enjoyed being able to not only work on the Site Designer product, but also to work with the tool on a daily basis to incorporate valuable insight about what we could do to improve the user experience of creating a site through Houzz. This process stretched my ability to think from the perspective of the user as well as the company to form solutions that would ulitmately benefit both parties. These improvements to the Website Services workflow, as well as to the Site Designer itself, are going to pivot the experiences for both the Pro and Houzz representative to a much more simple and elegant solution.
